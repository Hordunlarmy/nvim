# Changelog

## [Unreleased]

### Added
- Added file-save confirmation in the statusline ticker
- Added SQL language support with `sqls` LSP server
- Added `MasonLog` command to view Mason logs in a floating window
- Added new keymaps for notification history (`<leader>uh`) and last message (`<leader>uH`)
- Added mouse scroll support in WhichKey windows
- Added statusline guard to prevent invalid characters from breaking the UI
- Added manual progress state control via `progress.set_manual()` and `progress.clear_manual()`
- Added SQL filetype support in Treesitter and Aerial
- Added `statusline_guard` module to sanitize statusline content

### Changed
- Changed floating windows to be scoped to the current window instead of the full editor
- Changed notification system to use a custom ticker with better statusline integration
- Changed Aerial to auto-open only once per tab
- Changed progress bar to show more detailed information and support manual states
- Changed LSP hover windows to respect current window dimensions
- Changed Telescope pickers to use cursor theme with consistent sizing
- Changed notification history to use Noice's copyable split view
- Changed statusline to show progress bar when busy and ticker messages when idle
- Changed floating window positioning to be relative to current window
- Changed buffer-local floating windows (like `<leader>do` and `<leader>dO`) to be window-scoped

### Fixed
- Fixed floating windows appearing outside visible area
- Fixed notification dismissal not clearing the ticker
- Fixed statusline breaking on invalid characters
- Fixed Aerial opening multiple times in the same tab
- Fixed progress bar not updating correctly for LSP requests
- Fixed floating windows not respecting window splits
- Fixed notification history not being accessible when Noice is loaded
- Fixed statusline not updating when plugins are reloaded

### Removed
- Removed direct sourcing of `init.lua` during reload (now uses module reloading)
- Removed redundant notification history implementation in favor of Noice

### Security
- No security-related changes in this update
